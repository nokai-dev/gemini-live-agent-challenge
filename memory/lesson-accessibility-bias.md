# Hackathon Analysis Lesson — Accessibility Bias

**Date:** 2026-03-15
**Hackathon:** Gemini Live Agent Challenge
**Mistake:** Over-indexed on accessibility focus

## What Happened

I analyzed past winners (Jayu, Gaze Link, VITE VERE, SurgAgent) and noticed they all had accessibility angles. I then **assumed** the current hackathon required/rewarded accessibility focus.

**This was wrong.**

## The Error

1. **Past winners were accessibility-focused** → Gaze Link (ALS), VITE VERE (cognitive disability)
2. **I assumed this was required** → Built entire strategy around empathy engines
3. **I didn't verify** → Current hackathon examples are general: translator, tutor, customer support, marketing generator, workflow automator

## Root Cause

**Correlation ≠ Causation**
- Past winners happened to be accessibility projects
- This doesn't mean future winners must be accessibility projects
- The judging rubric rewards "breaking text box paradigm" — not "helping disabled people"

## Verification Checklist (Use for Future Hackathons)

Before defaulting to accessibility/empathy focus:

- [ ] Read the hackathon's **stated examples** — are they accessibility-focused?
- [ ] Check if the hackathon has **explicit accessibility tracks or prizes**
- [ ] Verify if past winners' accessibility angle was **required or coincidental**
- [ ] Ask: "Would a workflow eliminator or magic illusion win this too?"
- [ ] Check if the **sponsor's examples** include accessibility (Google's examples didn't)

## Current Hackathon Reality

**Gemini Live Agent Challenge actual examples:**
- Live Agents: "real-time translator, vision-enabled tutor, customer support voice agent"
- Creative Storyteller: "marketing asset generator, social content creator"
- UI Navigator: "workflow automator, QA testing agent"

**None require accessibility.** Accessibility is ONE valid angle, not THE angle.

## Corrected Approach

**Three valid archetypes for this hackathon:**
1. **Workflow Eliminator** — Universal web navigator, cross-app automation
2. **Illusion of Magic** — Real-time translator with barge-in, AI game opponent
3. **Empathy Engine** — Accessibility (optional, not required)

## Prevention

**Always verify:**
- What does THIS specific hackathon reward?
- What examples do THEY give?
- Are past winners' themes coincidental or required?

**Never assume** past winner patterns = current requirements.

---

*Lesson: Accessibility is powerful when appropriate, but not universally required. Verify the rubric, don't project patterns.*
