
## Continuous Improvement Cycle - 2026-03-21 20:04 UTC

### Critical Bug Found: Dockerfile Missing Static Files

**Issue:** The Cloud Run deployment would FAIL because the Dockerfile only copied `backend/src` but:
- Cloud Run env sets `STATIC_FILES_PATH=/app/static` (frontend built files)
- Cloud Run env sets `DEMO_PROJECT_PATH=/app/demo-project`
- **Neither directory existed in the container!**

**Root Cause:**
1. Build context was `./backend` but needed to be `.` to access `frontend/` and `demo-project/`
2. Dockerfile never copied frontend/dist/renderer
3. Dockerfile never copied demo-project/

**Fix Applied (commit d432e39):**
1. Changed Docker build context from `./backend` to `.`
2. Added frontend build step in deploy.yml (npm ci + npm run build)
3. Added COPY commands for frontend/dist/renderer and demo-project
4. Added ENV vars STATIC_FILES_PATH and DEMO_PROJECT_PATH to Dockerfile
5. Updated file paths in Dockerfile to use backend/ prefix
